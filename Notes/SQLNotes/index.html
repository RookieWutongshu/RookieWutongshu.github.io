Intro

  • SQL(Structured Query Language)指结构化查询语言
  • SQL使我们有能力访问数据库
  • SQL使一种ANSI( America National Standard Institute美国国家标准化组织)的标准计算机语言

What can SQL Do?

  • 创建新的数据库及其对象(表，索引，视图，存储过程，函数和触发器)
  • 从数据库中，执行查询，取回数据，插入数据，更新数据，删除数据
  • 可以Commit数据字典
  • RENAME一个对象
  • 在数据库中设置用户Grant和Revoke权限

RDBMS

Relational Database Management System.关系型数据库。RDBMS的数据都存于数据库对象——Table(表)中。表是相关数据项的集合，由列和行组成。每个表由更小的实体Field(字段)组成，字段是表中的一列，以maintain(维护)表中每条记录的特定信息。

record(记录)（也称为行），是存在于表中的每个单独entry(条目)，一条record是表中的一个horizontal entity(横向实体)。

column(列)是表中的vertical entity(垂直实体)，contains associated with 表中一个特定字段所有信息。

Syntax

Database Tables

Each table is identified by a name. Tables contains(包含) records(rows) with data.

SQL Statements(语句)

Most of the actions（操作） you need to perform on a database are done with SQL statements.

Keep in mind that

  • SQL keywords are not case sensitive(大小写不敏感)
  • Semicolon(分号) after SQL Statements
  • SQL statements always begin with keywords

For the second point,虽然不是每种DBS都要求，但他是种标准分割SQL语句的方法，让我们在 same call （同一请求）中执行多条语句

Some of the most important SQL Commands（命令）

  • SELECT - extracts(提取) data from a database
  • UPDATE - updates data in a database
  • DELETE - deletes data from a database
  • INSERT INTO - inserts new data into a database
  • CREATE DATABASE - creates a new database
  • ALTER DATABASE - modifies a database
  • CREATE TABLE - creates a new table
  • ALTER TABLE - modifies a table
  • DROP TABLE - deletes a table
  • CREATE INDEX - creates an index (search key)
  • DROP INDEX - deletes an index

The SQL Select Statement

The select statement is used to selcet data from a database.

The Data returned is stored（存储）in a result table,called the result-set

Seclect Syntax

select column1,column2,... from table_name;

if you want to select all the fields available（可获得的）in the table,use the following syntax:

select * from table_name;

Navigation（导航） in result-set

most DBS allow use 编程函数 to navigate in result-set 中,for example: Move-To-First-Record,Move-To-Next-Record,Get-Record-Content等等

SQL select distinct statement

The SQL SELECT DISTINCT Statement

The select distinct statement is used to return only distinct(独特的，different) values

Insides a table,a column often contains many duplicate (重复的) values; and sometimes you only want to list the distinct values.

SELECT DISTINCT Syntax

SELECT DISTINCT column1,column2,... FROM table_name;

The following SQL statement lists the number of different customer countries

SELECT COUNT(DISTINCT country) FROM Customers;

Note:The example above will not work in Firefox and Microsoft Edge Because they are using Microsoft Access Database.And this usage(用法) are not supported for it.We can use following statement to replace

SELECT COUNT(*) AS DistinctCountries FROM (SELECT DISTINCT Country FROM Customers);

SQL WHERE Clause(查询子句)

The Where clause is used to filter records,extract only those records that fulfill(满足) a specific condition.

WHERE Syntax

SELECT column1,column2,... FROM table_name WHERE condition;

WHERE Clause Example

SELECT * FROM Customers WHERE Country = 'Mexico';

Text Fields vs Numeric(数字，数值的) Fields

SQL requires single quotes(单引号) around text values(most database system will also allow the dobule quotes 双引号). However,numric fields should not be enclosed(被包围) in quotes:

SELECT * FROM Customer WHERE CustomerID = 1;

Operators(运算符) in The WHERE Clause

The following operators can be used in the WHERE clause:

Operator Description
= Equal
<> Not equal. Note: In some versions of SQL this operator may be written as !=
> Greater than
< Less than
>= Greater than or equal
<= Less than or equal
BETWEEN Between a certain range(确定的范围)
LIKE Search for a pattern（模式）
IN To specify(指定) multiple possible values for a column

SQL AND,OR and NOT Operators

The WHERE Clause can be conbined(结合) AND,OR,and NOT operators.

The AND and OR operators are used to fileter records based on more than one condutions.

The AND(OR) opetators displays a record if all(any) the conditons separated(分开) by AND are(is) TRUE.

The Not operators display a record if the condition(s) is NOT TRUE.

Syntax

SELECT column1,column2,... FROM table_name WHERE condition1 AND condition2 AND condition3...;
SELECT column1,column2,... FROM table_name WHERE condition1 OR condition2 OR condition3...;
SELECT column1,column2...FROM table_name WHERE NOT condition;

Combining(使结合) AND,OR and NOT

You can conbine the AND,OR and NOT operators.Use parenthesis[pə'rɛnθəsɪs 圆括号] to form(组成) complex expressions. 

SELECT ＊ FROM Customers WHERE Country='Germany' AND (City='Berlin' OR City='Muchen');
SELECT ＊ FROM Customers WHERE NOT Country='Germany' AND NOT Country='USA';　

SQL ORDER BY Keyword(按关键字排序)

The SQL ORDER BY keyword is used to sort(排序) the result-set in ascending or descending order(整理,顺序 ).

It sort the records in ascending order by default.To sort the records in descending order，use the DESC keyword.

ORDER BY Syntax

SELECT colomn1,column2... FROM table_name ORDER BY column1,column2,... ASC | DESC;

ORDER BY Several Columns Example

The following statement will sort the result-set order by Country,but if some rows have the same country,it orders them by CustomerName

SELECT * FROM Customers ORDER BY Country,CustomersName (ASC);

ORDER BY Several Columns Example2

sorted them in ascending order by the "Country" and descending by the "CustomerName" column

SELECT * FROM Customers ORDER BY Country ASC,CustomerName DESC;

SQL INSERT INTO Statement

The INSERT INTO statement is used to insert new records in a table.

INSERT INTO Syntax

INSERT INTO table_name(column1,column2,...) VALUES(values,values2,...);

If you are adding values for all the columns of the table,you do not need to specify the column names in the SQL statement.However,make sure the order of the values is in the same as the columns in the table.

INSERT INTO table_name VALUES (value1,value2,...);

The auto-increment(增量) field will be generated(生成) automatically when a new record is inserted into the table.

Of course,you can also possible to only insert data in specific columns.

SQL NULL Values

A field with NULL value is a field with no value.

If a field in table is optional(可选择的)，it is possible to insert or update a record wirhout adding a value to this field.Then,the field will be saved with a NULL value.

Note:A NULL value is different fron a zero value or a field that contains(包含) spaces.A field with a NULL value means it has been left blank(空白的) during record creation.

Syntax

Use IS NULL to test empty values.Use IS NOT NOT NULL test non-empty values

SELECT column1 FROM table_name WHERE column2 IS NULL;
SELECT column1 FROM table_name WHERE column2 IS NOT NULL;

When you create a table contains a field allowed to left blank,you can write as bellow

 CREATE TABLE CUSTOMERS(
   ID   INT              NOT NULL,
   NAME VARCHAR (20)     NOT NULL,
   AGE  INT              NOT NULL,
   ADDRESS  CHAR (25) ,
   SALARY   DECIMAL (18, 2),       
   PRIMARY KEY (ID)
);

SQL UPDATE Statement

UPDATE Statement is used to modify the existing(现存) records in a table.

UPDATE Syntax

UPDATE table_name SET column1 = value1,column2 = value2... WHERE condition;
--update all the records
UPDATE table_name SET column1 = value1...;

NOTE: Be careful when updating records in a table!The WHERE Clause is used to specify which record(s) should be updateed.If you omit(省略) it ,all records in the table will be updated!

SQL DELETE Statement

The DELETE statement is used to delete existing records in a table.

DELETE Syntax

DELETE FROM table_name WHERE condition;
-- delete all the records
DELETE FROM table_name;

NOTE:The same as UPDATE statement.Be careful the WHERE clause,if you omit WHERE clause,all records in the table will be deleted.

Why don't we drop the table directly? Because delete all the records can leave table structs,property and indexs.

SQL TOP, LIMIT or ROWNUM Clause

The SELECT TOP clause is used to specify the number of records to rerurn.

The SELECT TOP clause is useful on large tables with thousands of records.Returning a large number of records can impact(影响) on performance(性能).

Note:Not all database systems support the SELECT TOP clause.MySQL supports the LIMIT clause to selcet a limited number of records,whlie Oracle use ROWNUM.But MySQL and Oracle do not support percents as value,you can write a extra SQL statement to realize it such as SELECT COUNT(*)*percent FROM table_name WHERE condition;

SQL TOP Syntax

-- SQL Server / MS Access Syntax
SELECT TOP number | number PERCENT column1,column2... WHERE condition;
-- MySQL Syntax
SELECT column1,column2... FROM table_name WHERE condition LIMIT  number;
-- Oracle Syntax
SELECT colomn1,column2... FROM table_name WHERE conditon AND ROWNUM <= number;

SQL MIN() and MAX() Functions(函数)

The MIN() function returns the smallest value of the selected column.

The MAX() function return the lagerest value of the selected column.

MAX() and MIN()Syntax

SELECT MAX(column_name) FROM table_name WHERE codition;
SELECT MIN(column_name) FROM table_name WHERE conditon;

SQL COUNT(), AVG(),SUM() Functions

The COUNT() function returns the number of rows that matches(匹配) a specified criteria(条件).

The AVG() function returns the average(平均的['ævərɪdʒ]) value of a numeric column.

The SUM() function returns the total sum of a numric column.

Syntax

SELECT COUNT(column_name) FROM table_name WHERE condition;
SELECT AVG(column_name) FROM table_name WHERE condition;
SELECT SUM(column_name) FROM table_name WHERE condition;

such as SELECT COUNT(customerID) FROM Customer;

Note:The NULL values will be ignored.

Example

-- count the number of different customers in the oreders' table
SELECT COUNT(DISTINCT CustomerID) AS NumberOfCustomer FROM Orders;
-- The following statement choose the product's price that higher than average price
SELECT ProductName,price FROM Products WHERE price > (SELECT AVG(PRICE) FROM Products);
-- Add all quantity
SELECT SUM(Quantity) from Products;

SQL LIKE Operator

The LIKE operator is used in a WHERE clause to serach a specified pattern(模式) in a column.

There are two wildcards(通配符) often used in conjunction(结合，连词) with the LIKE operator;

  • % pecent sign represent zero, one, or multiple characters.
  • _ The underscore(连字符，下划线) represents a single character.

Notes: MS Access use asterisk(*) (星号 ['æstərɪsk]) instead of the percent sign(%), use question mark(?) instead of underscore(_).

The percent sign and the underscore can also be used in conbinations(组合)!

LIKE Syntax

SELECT  column1,column2,... FROM table_name WHERE columnN LIKE pattern;

Here are some examples showing different LIKE operators with ' % ' and ' _ ' wildcards:

LIKE Operator Description
WHERE CustomerName LIKE 'a%' Finds any values that start with "a"
WHERE CustomerName LIKE '%a' Finds any values that end with "a"
WHERE CustomerName LIKE '%or%' Finds any values that have "or" in any position
WHERE CustomerName LIKE '_r%' Finds any values that have "r" in the second position
WHERE CustomerName LIKE 'a_%_%' Finds any values that start with "a" and are at least 3 characters in length
WHERE ContactName LIKE 'a%o' Finds any values that start with "a" and ends with "o"

The SQL IN Operator

The IN operator allow you to specify multiple values in a WHERE clasuse.

The IN operator is a shorthand(简写，速记) for multiple OR condition.

Syntax

SELECT column_name(s) FROM table_name WHERE column_name IN  (value1,value2...);
SELECT column_name(s) FROM IN (SELECT STATEMENT);

NOTE: text value should be enclosed in single quote ;

Example

SELECT * FROM Customers WHERE country (NOT) IN ('USA','France');
-- The following statement will select all the customer that are from the countries as the suppliers(供应商):
SELECT * FROM Customers WHERE country IN (SELECT Country FROM Suppliers);

SQL BETWEEN Operator

The BETWEEN operator selects values within a given range.The values can be text, number or dates.

The BETWEEN operator is inclusive(包含性的): begin and end values are included.

Syntax

SELECT column_name(s) FROM table_name WHERE column_name between value1 AND value2;

Example

-- To display the product outside the range of the previous example,use NOT BETWEEN:
SELECT * FROM Products WHERE Price NOT BETWEEN 10 AND 20;
-- Select all product with price BETWEEN 10 and 20,In addition(另外); do not show priducts with a CategoryID of 1, 2, or 3:
SELECT * FROM Product WHERE (Price BETWEEN 10 AND 20) AND (CategoryID NOT IN (1,2,3));
-- NOT operator also can be written before the CategoryID,it make the same effect.
-- The following statement select all orders with OrderDate between '01-July-1996' and '31-July-1996':
SELECT * FROM Orders WHERE OrederDate BETWEEN #01/07/1996# AND #31/07/1996#;
--Or the bellow(above syntax are not supported in MySQL)
SELECT * FROM Orders WHERE '1996-07-01' AND '1996-07-31';

Tips:In diffrerent DBSM, the BETWEEN oprator will be produce(产生) different results.

Such as BETWEEN 1 AND 10; It may be regarded as (1,10) or [1,10] or [1,10).

So please check your how your DB deal with BETWEEN operator.

SQL Aliases(别名)

SQL aliases are used to give a table, or a column in a table, a temporary name.

Aliases are often be used to make column name more readable.

An alias only exits for the duration of the query.

Syntax 

SELECT column_name AS alias_name FROM table_name;
SELECT column_name(s) FROM table_name AS alias_name;

Note: If an alias name contains space, it requires double quotation mark(双引号) or square brackets(方括号).

MySQL does not supports square brakets for Aliases.

Example

SELECT CustomerName AS Customer,CntactName AS [Contact Person] FROM Customers;
-- Your can create a alies named "address" that combine four colomns(Address,PostalCode,City and Country)
SELECT CustomerName, Address + ',' + PostalCode + ' ' + City + Country AS Address FROM Customers;
-- In MySQL you should write as following:
SELECT CustomerName,CONCAT(Address,','PostalCode,', 'City,', ',Country) AS Address FROM Customers;
SELECT o.OrderID, o.OrderDate, c.CustomerName
FROM Customers AS c, Orders AS o
WHERE c.CustomerName="Around the Horn" AND c.CustomerID=o.CustomerID;

Alies can be useful when:

  • There are more than one table involved(涉及) in a query
  • Functions are used in the query
  • Column names are big or not very readable
  • Two or more columns are combined together

SQL Joins

A JOIN clause is used to combine rows from two or more tables, based on a related column between them.

Different type of SQL JOINS

  • (INNER) JOIN:Returns record that have matching values in both.
  • LEFT (OUTER) JOIN: Return all records from the left table, and the matched records from the right table.
  • RIGHT(OUTER) JOIN:Return all records from the right table,and the matched records from the left table.
  • FULL(OUTER) JOIN:Return all records when there is a match in either left or right table.
  • CARTESIAN JOIN(笛卡尔连接)：Return result-sets' cartesian product(笛卡尔积) from two or more tables.

Cartesian join as the same as the INNER JOIN without WHERE clause.

Syntax

SELECT column_name(s) FROM table1 INNER JOIN table2 ON table1.column_name = table2.column_name;
SELECT column_name(s) FROM table1 LEFT JOIN table2 ON table1.column_name = table2.column_name;
SELECT column_name(s) FROM table1 RIGHT JOIN table2 ON table1.column_name = table2.column_name;
SELECT column_name(s) FROM table1 FULL JOIN table2 ON table1.column_name = table2.column_name;
SELECT column_name(s) FROM table1,table2,...;

Note:Some database do not support full join, you can use UNION ALL to combine the left join and right join.

SELECT column_name(s) FROM table1 LEFT JOIN table2 ON table1.column_name = table2.column_name
UNION ALL
SELECT column_name(s) FROM table1 RIGHT JOIN table2 ON table1.column_name = table2.column_name;

Example

-- The follwing SQL statement select all orders with customer infomation:
SELECT a.OrderID,b.CustomerName FROM Orders AS a INNER JOIN Customers AS b ON a.CustomerID = b.CustomerID;
/*
JOIN Three tables
The follwing SQL statement select all orders with customer and shipper information:
*/
SELECT a.OrderID,b.CustomerName,c.ShipperName FROM( (Orders AS a INNER JOIN Customers AS b ON a.CustomerID = b.CustomerID) INNER JOIN Shippers AS c ON a.ShipperID = c.ShipperID);

SQL Self JOIN

A self JOIN is a regular(常规) join, but the table is joined with itself.

Syntax

SELECT column_name(s) FROM table1 T1,table T2 WHERE condition;

Example

-- The following SQL statement matches customers that from the same city:
SELECT a.CustomerName AS customer1,b.customerName AS customer2,a.city FROM Customers a,Customers b WHERE a.CustomerID <> b.CustomerID AND a.city = b.city ORDER BY a.city;

SQL UNION Operator

The UNION operator is used to combine the result-set two or more SELECT statement.

  • Each SELECT statement within UNION operator must have the same number of columns
  • The columns must also have similar data types.
  • The columns in each SELECT statement must also be in the same order.

The Union result-set's column name always as the name as the column name in the first select statement.

Syntax

SELECT column_name(s) FROM table1 UNION SELECT column_name(s) FROM table2;

The UNION operator selects only distinct values by default.To allow duplicate values, use UNION ALL:

SELECT column_name(s) FROM table1 UNION ALL SELECT column_name(s) FROM table2;

Example

--returns the German cities(duplicate values also) from both the "Customers" and "Suppliers" table:
SELECT City,Country FROM Customers WHERE Country = 'German'
UNION ALL
SELECT City,Country FROM Suppliers WHERE Country = 'German'
ORDER BY City;
-- list all customrts and suppliers
SELECT 'Customer' AS Type,  CustomerName FROM Customers
UNION ALL
SELECT 'Suppliers',SupplierName FROM Suppliers;

SQL GROUP BY Statement

The GROUP BY statement is often used with aggregate(聚合) functions(COUNT, MAX, MIN, AVG, SUM) to group the result-set by one or more columns.

Syntax

SELECT column_name(s) FROM table_name WHERE condition GROUP BY column_name(s) ORDER BY column_name(s);

Example

-- list the number of customers from the each country,sorted high to low
SELECT COUNT(CustomerID), Country FROM Customers GROUP BY Country ORDER BY COUNT(CustomerID) DESC;
-- list the number of orders sent by each shipper:
SELECT COUNT(o.OrderID),s.ShipperName FROM Orders AS o LEFT JOIN Shippers AS s ON o.ShipperID = s.ShipperID;

Note:Remember what condition you want to sort the record, do not fogot to add the aggregate functions.

SQL HAVING Clause

The HAVING clause was added to SQL because the WHERE keyword could not be used with aggregate functions.

Syntax

SELECT column_name(s) FROM table_name WHERE conditon GROUP BY column_name(s) HAVING condition ORDER BY column_name(s);

Example

-- list the number of customers in each country.Only include countries with more than 5 customers:
SELECT COUNT(CustomerID),Country FROM Customers GROUP BY Country HAVING COUNT(CustomerID) >5;
-- list if the employees "Mike" or "Sara" have registered more than 25 orders:
SELECT e.LastName,COUNT(o.OrderID) FROM Employees AS e INNER JOIN Orders AS o ON e.EmployeeID = o.EmployeeID  
WHERE e.LastName IN ('Mike','Sara')
GROUP BY e.LastName
HAVING COUNT(o.OrderID) > 25;

What's the difference between ON, WHERE, HAVING clause

Judge(判断) from the excute orders, their excute order is ON -> WHERE -> HAVING.

In the case of(在...的情况下) LEFT JOIN ,ON condition is used in creating temporary table, it doesn't mattter if the condition is true, it will show all the records from the left table. However, the WHERE contion is used after the temporary table is created , then filter the record that condition is false in the temporary table. So if you use WHERE operator, it is truly as the same as use INNER JOIN.

In multi-table join query, DBMS will create a temporary table base on ON condition, then filter by WHERE clause, and then do some aggregate function or others, after calculate(计算), filter by HAVING clause.

SQL EXISTS Operator

The EXISTS operator is used to test the existence of any record in a subquery(子查询).

The EXISTS operator returns true if the subquery returns one or more records.

Syntax

SELECT column_name(s) FROM table_name WHERE EXISTS (SELECT column_name)

Note: Why the second "table_name" does not exists "(s)", although the tutorial does not refer it, through my test in mysql, the subquery will return all the columns that match the condition. So you don't need to specify the columns that you want to use in the parent query.

Example

-- The following statement list the suppliers with a product price less than 20:
SELECT SupplierName FROM Suppliers WHERE EXITS(SELECT ProductName FROM Products WHERE Products.SupplierID = Suppliers.ID AND Price < 20 );

SQL ANY and ALL Operator

The ANY and ALL operator are used with a WHERE or HAVING clause.

The ANY operator returns true if any of the subquery values meet the condition.

The ALL operator returns true if all of the subquery values meet the condition.

Do not be confused with ANY and EXISTS operator

Syntax

SELECT column_name(s) FROM table_name WHERE column_name operator ALL | ANY 
(SELECT column_name FROM table_name WHERE condition);

Note: The operator must be a standard comparision(比较) operator (=, <>, !=, >, >=, <, or <=) .

Example

-- The follwing statement will lists the productnames if it finds any records in the OrderDetails table that quantity(数量) = 10;
SELECT ProductName FROM Products WHERE ProductID = ANY (SELECT ProductID FROM OrderDetails WHERE Quantity = 10);

SQL SELECT INTO Statement

The SELECT INTO statement copies data from one table into a new table.

Syntax

SELECT * INTO newtable [IN externaldb] FROM oldtable WHERE condition;

external:外部的

SELECT column1,column2...INTO newtable [IN externaldb] FROM oldtable WHERE condition;

Example

SELECT * INTO CustomersBackup2019 IN 'Backup.mdb' FROM Customers; 
SELECT c.CustomerName,o.OrderID INTO CustomersBackup2019 IN 'Backup.mdb' FROM Customer AS c LEFT JOIN Orders AS o ON c.CustomerID = o.CustomerID;

Tips: You can also create a new empty table using the schema(模式) of another, Just add a WHERE clause that cause the query to return no data

SELECT * INTO Customer2019 FROM Customer2018 WHERE 1 = 0;

However, MySQL do not support SELECT INTO statementm, you can use the following statement

-- Copy data
CREATE TABLE CustomersBackup(SELECT * FROM Customers);
-- Only copy table schema
CREATE TABLE CustomerBackup LIKE Customers.

SQL INSERT INTO SELECT Statement

The INSERT INTO SELECT statement copies data from one table and insert it into anther table.

  • INSERT INTO SELECT requires that data types in source and target tables match
  • The existing records in the target table are unaffected.

Syntax

INSERT INTO table2 SELECT * FROM table1 WHERE condition;
INSERT INTO table2 (column1,column2...) FROM table1 WHERE condition;

SQL CASE Statement

The CASE statement goes through conditions and return a value when the first comdition is met(like an IF-THEN-ELSE statement). So, once a condition is true, it will stop reading and return the result. If no conditions are true, it returns the value in the ELSE clause.

If there is no ELSE part and no codition are true, it returns NULL;

Syntax

CASE
    WHEN condition1 THEN result1
    WHEN condition2 THEN result2
    WHEN conditionN THEN resultN
    ELSE result
END;

Example

SELECT OrderID,Quantity,
CASE
    WHEN Quantity > 30 THEN "The quantity is more than 30"
    ELSE "The quantity is less or equal 30"
END AS Quantity
FROM Orders;
-- Another example
SELECT CustomerName,City,Country
FROM Customers
ORDER BY
(CASE 
    WHEN City IS NULL THEN Country
    ELSE City
 END);

SQL NULL Functions

Look at the statement below first

SCELECT ProductName, UnitPrice*(2 + UnitsOnOrder) FROM Products;

If any of the "UnitsOnOrder" value are NULL, the result will be NULL.

Solutions

The MySQL IFNULL() or COALESCE() (合并)function let you return an alternative value if an expression is NULL:

SELECT ProductName, UnitPrice * (1 + IFNULL(UnitsOrder,0)) FROM Orders;
SELECT ProductName, UnitPrice * (1 + COALESCE(UnitsOrder,0)) FROM Orders;

In SQL Server use ISNULL() while MS Access use NVL() to solve this question

SELECT ProductName, UnitPrice * (1 + ISNULL(UnitsOrder,0)) FROM Orders;
SELECT ProductName, UnitPrice * (1 + NVL(UnitsOrder,0)) FROM Orders;
